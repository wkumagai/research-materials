## エグゼクティブサマリー

AIXSが構築すべきMoatを規制・エコシステム・市場構造・文化の4層で分析した結果、**最も強力で持続的なMoatは「規制・認証の積層」と「公的研究エコシステムへの組み込み」の組み合わせ**である。一方、単独のMoatはいずれも3-5年以内に崩壊するリスクがあり、複数のMoatを重ね合わせる「堀の多層化」が不可欠である。

**最重要の結論:**
- 規制Moat単体の持続期間：24-36ヶ月（さくらインターネットが同様の動きをした場合は12-18ヶ月）
- エコシステムMoat単体の持続期間：18-36ヶ月
- 市場構造Moat単体の持続期間：36-60ヶ月（最も持続するが受動的）
- **3層を積層した場合の持続期間：48-72ヶ月（AIXSが唯一取りうるポジション）**

## 1. 規制・認証によるMoat

### Moat 1-1: ISMAP認証 + ガバメントクラウド連携

#### 最新状況（2026年3月時点）

- **ISMAPクラウドサービスリスト登録数:** 約77サービス（2025年6月時点）。2023年5月の44サービスから着実に増加しているが、GPU/AIコンピュート特化のサービスは極めて少ない
- **ISMAP管理基準の大改定（2025年12月公表）:** 詳細管理策が**約1,081項目から253項目に大幅削減**。ISO/IEC 27002に準じた5分類に再編。新基準施行は2027年3月以降の見込み
- **ガバメントクラウド:** 2026年3月27日、デジタル庁がさくらインターネットを**国内初のガバメントクラウドプロバイダとして正式認定**（305の技術要件を全て充足）。AWS、Google Cloud、Azure、Oracle Cloud Infrastructureに続く5社目
- **取得費用:** ISMAP本体で3,000-5,000万円、ISMAP-LIU（低影響用途）で1,000-3,000万円。監査期間は最短1年、最長1年9ヶ月

#### AIXSのMoat構築戦略

**直接ISMAP取得ではなく、さくらインターネットのISMAP認証済みインフラ上で動作する「ISMAPレディ」のポジションを取る。**

根拠:
1. スタートアップがISMAP認証を直接取得するのは費用（3,000-5,000万円）・期間（1-1.5年）の両面で非現実的
2. さくらインターネットが2026年3月にガバメントクラウド正式認定されたことで、さくらのインフラ上に構築されたサービスは「ガバメントクラウド上で動作するSaaS」として政府調達の対象になりうる
3. 2025年12月のISMAP管理基準改定で管理策が253項目に削減され、将来的な直接取得のハードルも低下傾向

**具体的なステップ:**
- Phase 1（M0-M6）: さくらのクラウド/高火力上にAIXSプラットフォームをデプロイ。「さくらISMAP認証インフラ上で動作」を明記
- Phase 2（M6-M18）: ISMAP-LIU（低影響用途）の取得準備開始。管理基準改定後の新基準（253項目）に準拠
- Phase 3（M18-M36）: ISMAP本体の取得、または「さくらガバメントクラウド上のマネージドサービス」として登録

**障壁の強度:** ★★★★☆
- 外資が直接ISMAPを取得するのは可能だが（AWS/Azure/GCPは取得済み）、**ISMAP + 日本語対応 + 科研費直接課金 + 年度予算対応**の組み合わせを構築するのは外資にとって非自明
- ただし、さくらインターネットが同様のMLプラットフォームを構築した場合、ISMAP面での優位性は消滅する

### Moat 1-2: 個人情報保護法改正 + 研究データガバナンス

#### 最新状況

- **個人情報保護法改正（2026年1月9日方針公表）:** 個人情報保護委員会が「3年ごと見直し」の制度改正方針を公表。主要論点に**クラウドサービスにおける越境移転規律の強化**と**生成AI学習段階の事業者義務の明確化**を含む
- **法案成立見込み:** 2026年5-6月頃。施行は法案成立から1-2年後（2027-2028年）
- **研究データに関する規律:** クラウドサービスやAIの実用化により、個人データに係るデータベースの構築やその処理プロセスについて、実質的には第三者（クラウド事業者）に依存するケースが拡大していることが問題視されている

#### AIXSのMoat構築戦略

**「研究データの国内完結保証」を差別化ポイントに据える。**

1. **医学・心理学研究の個人データ:** 被験者の個人情報を含む研究データは、改正個人情報保護法の越境移転規制が直接適用される。国内データセンターでの処理保証が競争優位に
2. **生成AI学習データの出自管理:** 研究機関が生成AIを学習させる際、学習データの出自管理・トレーサビリティが法的義務化される見込み。これはAIXSのプラットフォーム上で自動的にトラッキングする機能として実装可能
3. **文科省ガイドライン「研究機関における公的研究費の管理・監査のガイドライン」への準拠:** 公的研究費で利用するクラウドサービスには、適切な管理・監査体制が求められる

**障壁の強度:** ★★★☆☆
- 規制強化はAIXSにとって追い風だが、AWS/Azure/GCPも「日本リージョン」で対応可能
- 差別化は「規制対応の自動化」（コンプライアンスのビルトイン化）にある。外資はこれを日本市場のためだけに開発するインセンティブが低い

### Moat 1-3: EU AI Act対応 + ソブリンAI要件（将来のグローバル展開用）

#### 最新状況

- **EU AI Act施行スケジュール:**
  - 2025年2月: 禁止AI慣行の適用開始
  - 2025年8月: GPAI（汎用目的AI）のガバナンスルール・義務の適用開始
  - **2026年8月2日: 高リスクAIシステムの全面適用（最も重要な期限）**
- **クラウドプロバイダーへの影響:** クラウドプロバイダーはGDPR上の「データ処理者」であるだけでなく、AIシステムのトレーサビリティ・保存・セキュリティを保証する「エスコートオペレーター」としての役割が求められる
- **コンピュート報告義務:** システミックリスクを持つGPAIモデル（学習に10^25 FLOP超を使用したモデル）は、学習のエネルギー消費報告が義務化
- **罰則:** 2026年8月2日以降、違反には**最大3,500万ユーロまたはグローバル売上高の7%**の制裁金
- **EU Cloud and AI Development Act（CADA）:** 2026年Q1に提案予定。クラウド主権のさらなる法制化

- **ソブリンクラウド市場規模:** 2025年のUSD 1,547億から2034年にUSD 11,333億へ成長（CAGR 24.6%）
- **日本のソブリンクラウド動向:**
  - SoftBankがOracle Alloyで東西データセンターに国内完結ソブリンクラウドを構築（東日本2026年4月、西日本2026年10月開始）
  - GMI Cloudが日本でUSD 120億のソブリンAIインフラ投資を発表
  - 経済安全保障推進法に基づき、クラウドプログラムの安定供給計画が承認される動き

#### AIXSのMoat構築戦略

**Phase 3（M24-M36+）のグローバル展開時に、EU AI Act準拠を「ビルトイン」にした研究プラットフォームとして差別化。**

1. **コンピュートトレーサビリティ:** 研究で使用したGPU時間・エネルギー消費・データ所在地を自動記録。EU AI Act第10条（データガバナンス）と第12条（記録保持）への自動準拠
2. **AI Act研究免除の活用:** EU AI Actは研究目的のAIに免除規定がある。AIXSのR&D特化ポジションはこの免除を活用できる
3. **ソブリンデータ保証:** 日本国内データセンターでの処理保証は、日本政府の経済安全保障要件と、将来的なEU適十性認定のダブルミートに有利

**障壁の強度:** ★★★☆☆（現時点）→ ★★★★☆（2027-2028年、全面施行後）
- 現時点ではMoatとしての即効性は低い
- 2026年8月の全面施行後は、EU市場参入のためのコンプライアンスコストが参入障壁になる

### Moat 1-4: 学術出版社の再現性要件への準拠

#### 最新状況

- **Nature Portfolio:** 全研究論文にデータアクセシビリティ情報の記載を義務化。カスタム開発ソフトウェアは出版時にコードの公開が必須。Nature CommunicationsはCode Oceanとの連携で、査読プロセス中のコード共有・クラウド実行環境を提供開始
- **Springer Nature Code Policy:** 「Code Availability Statement」を全原著論文に義務化。新しいコードは論文結論の解釈・複製に必要な場合、公開が必須
- **ACM:** Reproducibilityバッジシステムを運用。ACM Multimedia 2025ではReproducibility Committeeが「Results Reproduced」バッジの取得を推奨。SIGMOD 2020-2025年にわたり再現性結果トラッキングを実施
- **IEEE:** 2026 World Congress on Computational Intelligenceで、採択時にデータ/コード公開を奨励

#### AIXSのMoat構築戦略

**「論文投稿時にワンクリックで再現性パッケージを生成」する機能は、既存分析でも指摘されたキラー機能だが、最新動向を踏まえるとさらに重要性が増している。**

1. **Code Ocean連携/競合:** Nature CommunicationsがCode Oceanと連携したことは、「実験の再現性にクラウド実行環境が必要」という市場認知が確立されたことを意味する。AIXSはCode Oceanの研究計算版（GPU/HPC対応版）として位置づけられる
2. **ACM Reproducibilityバッジ自動取得支援:** AIXS上で実験を行えば、ACMの「Results Reproduced」バッジに必要なアーティファクト（コード、データ、環境、実行ログ）が自動生成される仕組み
3. **出版社API連携:** Nature、ACM、IEEEの投稿システムとのAPI連携で、AIXS上の実験結果を論文投稿プロセスに直接埋め込む

**障壁の強度:** ★★★☆☆
- Code Oceanが直接的な競合。ただしCode Oceanは「コードの再実行」に特化しており、「GPU/HPC実験の全ライフサイクル管理」は射程外
- W&B、MLflowも同様の機能を追加可能だが、出版社との連携は研究ドメイン知識が必要

## 2. エコシステム・パートナーシップによるMoat

### Moat 2-1: ABCI / 公的研究基盤との連携

#### 最新状況

- **ABCI 3.0:** 2025年1月に3.0へ刷新。6,128基のNVIDIA H200 GPU搭載。2025年4月15日からサービス提供開始。ポイント単価220円/ポイント（税込）
- **ABCI利用者:** 約3,000人、566グループ（既存分析の数値と整合）
- **ABCI 2026年度継続利用:** 3月23日13:00より受付開始
- **富岳NEXT:** 理化学研究所、富士通、NVIDIAの国際連携により開発体制始動（2025年8月発表）。2030年頃稼働予定。アプリケーション実効性能で富岳比**最大100倍**。日本のフラッグシップシステムとして初めてGPUを採用。2025年度予算約73億円
- **mdx（データ活用社会創成プラットフォーム）:** 東京大学のmdx I、大阪大学のmdx IIで構成。全国の大学・研究機関・民間企業に提供。SINET経由でデータ収集・集約・処理機能を提供

#### AIXSのMoat構築戦略

**ABCIの「フロントエンドパートナー」ではなく、「ABCIを含むマルチ公的基盤のオーケストレーター」として位置づける。**

根拠:
1. ABCIは計算資源の提供に特化しており、実験管理・ワークフロー自動化・AIエージェント統合は提供していない
2. 研究者はABCI以外にもmdx、大学のローカルクラスタ、商用クラウドを使い分けており、これらを統合管理するニーズがある
3. 富岳NEXTの2030年稼働に向け、「AI for Science開発用テストベッド」の構築が進む。AIXSはこのテストベッドのフロントエンドとして関与する機会がある

**具体的パートナーシップ候補:**

| パートナー | 連携内容 | 期待効果 | 難度 |
|---|---|---|---|
| 産総研（ABCI運営） | ABCI APIとの統合、ジョブ管理UI提供 | 3,000人ユーザーベースへのアクセス | 高（公的機関の意思決定に6-12ヶ月） |
| 理研R-CCS | 富岳NEXT開発テストベッドとの連携 | 長期的な基盤ポジション確保 | 極めて高（2030年稼働に向けた長期計画） |
| NII（学認クラウド） | 学認IdP連携、SINET接続 | 全国950+大学へのシングルサインオン | 中（学認は既存のフェデレーション基盤） |
| 東大/阪大（mdx運営） | mdxとの計算資源連携 | 産学連携データプラットフォームとの統合 | 中-高 |

**障壁の強度:** ★★★★☆
- 公的研究基盤との連携は、**参入まで時間がかかるが、一度構築されると極めて強固**
- 外資クラウドがABCI APIを直接統合するインセンティブは低い（日本市場限定、規模が小さい）
- さくらインターネットはABCI 3.0の計算資源提供企業であり、このポジションを脅かす可能性がある

### Moat 2-2: GENIAC / J-Startup エコシステムへの組み込み

#### 最新状況

- **GENIAC第3期:** 2025年7月15日に24件の基盤モデル開発を採択。事業期間2025年8月-2026年2月。AWSがNVIDIA H100/H200 GPUを搭載したインスタンスを計算資源として提供
- **GENIACの今後:** 2025年12月26日公表（2026年1月9日改訂）の文書で今後の公募に関する検討状況を説明。スタートアップとユーザーが組んだモデル開発への支援を増やす方針。「エコシステム」としてモデル開発者、アプリ開発者、ユーザー企業、計算資源提供者、VC、SIerまで拡大中
- **J-Startup第5次選定（2025年3月）:** 新たに31社を選定、累計272社。Sakana AIが最大の資金調達額（約300億円）
- **METI FY2026 AI予算:** ¥1.23T（$7.9B、前年比4倍増）
- **科研費総額:** ¥237.9B/年（~$1.6B）

#### AIXSのMoat構築戦略

**GENIACの「計算資源提供者」側ではなく、「GENIAC採択プロジェクトの実験管理プラットフォーム」としてエコシステムに入り込む。**

1. **GENIAC採択24プロジェクトへのフリートライアル提供:** GENIAC採択プロジェクトは大量のGPU計算を行うが、実験管理は各自で行っている。AIXSの実験管理機能を無料で提供し、GENIAC内での認知度を獲得
2. **J-Startup認定の取得:** J-Startup認定はJETROの海外展開支援、政府のイベント参加、各種補助金優遇を受けられる。272社中のAI関連スタートアップは限定的であり、R&Dコンピュート特化のポジションは差別化可能
3. **NEDO DTSU/STS申請:** NEDO DTSU（最大1億円、費用の2/3補助）やSTS（最大7,500万円）を活用。公的資金の獲得自体がMoatではないが、**公的資金のプロジェクトで使われること自体が信頼のシグナル**になる

**障壁の強度:** ★★★☆☆
- J-Startup認定は取得すればMoatになるが、認定基準は不透明
- GENIACエコシステムへの参画は競争が激しい（AWSが既に計算資源提供者として深く入っている）
- さくらインターネットはGENIACの計算資源提供で既にエコシステム内にいる

### Moat 2-3: 学会・大学コミュニティとの組織的連携

#### 最新状況

- **学認クラウド:** NIIが運営する大学・研究機関のクラウド導入支援サービス。第40回「研究教育のためのクラウド利活用セミナー」を2026年1月-3月に開催
- **SINET6:** 全国950以上の大学・研究機関、300万人以上の研究者・学生が利用。国立大学カバー率100%、公立約95%、私立約71%
- **NII学位・資格情報デジタル化:** 2026年2月9日、NIIと野村総合研究所が学修成果のデジタル化に関する共同研究を開始
- **競争的研究費でのクラウド利用:** AWSブログ記事によると、科研費を含む競争的研究費でクラウド利用料の支出が可能であることが明確化されている

#### AIXSのMoat構築戦略

**「学認IdP連携 → 科研費直接課金 → 学会スポンサー」の三段階で、研究コミュニティに不可逆的に組み込まれる。**

1. **学認（GakuNin）IdP連携:** SINET6に接続されている950+大学・研究機関の認証基盤と連携。研究者は大学のアカウントでAIXSにログインでき、所属機関の確認が自動的に行われる。これにより、大学の既存認証インフラにAIXSが「乗る」形になり、個別の契約プロセスが不要に
2. **科研費直接課金対応:** 科研費の直接経費でクラウド利用料が支出可能であることは明確化されているが、予算管理機能（年度内執行管理、残高アラート、科研費費目別自動分類）を提供するプロバイダは存在しない。AIXSがこれを実装すれば、研究者の事務作業を大幅に削減
3. **主要学会との連携:**
   - JSAI（人工知能学会）: 年次大会でのワークショップスポンサー
   - 情報処理学会: 全国大会でのデモ展示
   - 電子情報通信学会: 研究会でのプレゼンテーション
   - NeurIPS/ICML/ICLR: 日本人研究者コミュニティへの重点アプローチ

**障壁の強度:** ★★★★☆
- 学認IdP連携は技術的には容易だが、**NIIとの公式パートナーシップ**が必要であり、一度構築されれば競合の参入障壁になる
- 科研費対応課金は外資にとって「日本市場のためだけに開発する」機能であり、優先度が低い
- 学会ネットワークは属人的な関係に依存し、再現が困難

## 3. 市場構造によるMoat

### Moat 3-1: 年度予算制度という構造的障壁

#### 日本の研究予算の特殊性

日本の大学・研究機関の予算制度は、外資クラウドプロバイダーにとって以下の構造的な障壁を生む:

1. **会計年度（4月-3月）:** 科研費を含む公的研究費は会計年度内での執行が原則。AWS/GCPの月額従量課金は年度末の予算消化・翌年度への繰越管理と相性が悪い
2. **予算の費目分類:** 科研費は「設備備品費」「消耗品費」「旅費」「人件費・謝金」「その他」に分類され、クラウド利用料は通常「その他」に分類される。しかし高額GPU購入は「設備備品費」。費目間の流用には制限がある
3. **機関管理の間接経費:** 科研費の直接経費に対して30%の間接経費が大学に配分される。大学にとってはクラウド利用料が多いほど間接経費が増えるインセンティブがある
4. **購入における入札制度:** 一定額以上の物品・サービスには入札が必要。クラウドサービスは「役務提供」として随意契約が可能な場合が多いが、大学ごとにルールが異なる
5. **共用義務（2025年度新規）:** 2025年度から、科研費で購入した研究設備・機器のうち一定条件を満たすものは研究機関内外への共用が求められる

#### AIXSの対応策

- **科研費費目自動分類:** GPU利用料を費目別（設備備品費 vs その他）に自動分類し、科研費報告書フォーマットでの出力を可能にする
- **年度予算管理ダッシュボード:** 会計年度（4月-3月）に対応した予算消化率、残高、月別コスト推移を可視化
- **プリペイド/ポイント制:** ABCIのポイント制（220円/ポイント）に倣い、年度初めにポイントを一括購入し、年度内で消化する仕組み。大学の調達プロセスと整合的
- **見積書/請求書/納品書の日本語フォーマット:** 大学の会計部門が求める書類を日本語で自動生成

**障壁の強度:** ★★★★★（日本市場最強のMoat候補）
- 年度予算制度は法制度に根ざしており、3-5年で変わるものではない
- AWS/GCPは日本の年度予算制度に完全対応する機能を開発するインセンティブが低い（グローバル顧客の0.1%未満のニーズ）
- さくらインターネットは国内企業としてこの対応は可能だが、現時点では研究者向けの予算管理機能は提供していない

### Moat 3-2: 共同利用・共同研究体制という独自市場構造

#### 日本の研究体制の特殊性

日本の大学は「共同利用・共同研究拠点」制度により、特定の研究資源を全国の研究者が共同で利用する体制が制度化されている:

- **大学共同利用機関法人:** 高エネルギー加速器研究機構（KEK）、国立天文台、国立情報学研究所（NII）等。これらの機関が保有する計算資源は全国の研究者に開放されている
- **共同利用・共同研究拠点:** 文科省認定。全国の研究者が所属機関にかかわらず利用できる研究拠点
- **学際大規模情報基盤共同利用・共同研究拠点（JHPCN）:** 東大、東工大、名古屋大、京大、阪大等9大学のスーパーコンピュータを共同利用するネットワーク

この「共同利用体制」は、**個別の大学が独自にクラウドを契約するのではなく、拠点が一括でサービスを導入し、全国の研究者に提供する**というモデルを意味する。AIXSにとっては、1つの拠点との契約で数百-数千の研究者にリーチできる可能性がある。

**障壁の強度:** ★★★★☆
- 共同利用体制は日本独自の制度であり、外資がこの構造を理解して営業する難度は高い
- 一度拠点に採用されれば、全国の研究者に波及するレバレッジ効果がある
- ただし、拠点の意思決定は委員会制で遅く（6-18ヶ月）、初期のスピードが犠牲になる

## 4. 地理的・文化的Moat

### Moat 4-1: 「日本語」を超えた研究文化の特殊性

日本語対応は表面的なMoatに過ぎない。真に深い文化的障壁は以下にある:

#### 4-1-1: 「根回し」と合意形成プロセス

日本の大学・研究機関では、新しいツールの導入に以下のプロセスが必要:
1. **研究室単位の非公式合意:** PI（主任研究者）だけでなく、技術職員・事務職員・学生の間で非公式な合意形成が行われる
2. **学科/専攻レベルの合意:** 情報基盤委員会等での議論を経て、組織的な導入が決定される
3. **事務局・会計部門との調整:** 支払方法、予算費目、契約形態についての事前調整が必要
4. **年度計画への組み込み:** 多くの場合、翌年度の予算計画に組み込む形で導入される

外資クラウドの「サインアップして即利用」モデルは、この合意形成プロセスと根本的に相性が悪い。AIXSが「研究室のボスにメールで説明→技術職員が試用→事務局に見積もり提出→年度計画に組み込み」というフローをサポートできれば、強力な文化的Moatになる。

#### 4-1-2: 「技術職員」という隠れた意思決定者

日本の大学研究室には「技術職員」（テクニシャン/技官）が存在し、実験装置の管理、計算機環境の構築、データ管理を担当している。この層は:
- 学会論文には名前が出ないが、ツール選定の実質的な決定権を持つ
- 英語のドキュメントを読むことはできるが、日本語サポートを強く好む
- 安定性と実績を重視し、新しいツールの導入には慎重

AIXSが「技術職員向けの管理者ダッシュボード」（利用状況モニタリング、コスト管理、ユーザー管理）を充実させることで、この隠れた意思決定者を味方にできる。

#### 4-1-3: 学会文化と「知り合い」ネットワーク

日本の研究コミュニティはグローバルに比べて閉鎖的であり:
- 新しいツールは学会での口コミで広まる
- 主要研究室のPIが使っているツールが「デファクト」になる
- 海外ツールよりも「知り合いが作ったツール」の方が信頼される

AIXSの創業チームが研究コミュニティの出身であることは、この文化的Moatの構築に不可欠である。

#### 4-1-4: 「報告書文化」と形式的要件

日本の研究機関では、報告書・申請書の形式が厳密に定められている:
- 科研費の実績報告書（JSPS指定フォーマット）
- NEDO/JST事業の中間・最終報告書
- 大学の年次研究活動報告書

AIXSが「プラットフォーム上の活動データから報告書を自動生成」する機能を持てば、研究者の事務作業負担を大幅に軽減でき、強力な差別化要因になる。

**障壁の強度:** ★★★★★（最も深い文化的Moat）
- 根回し文化、技術職員の存在、学会ネットワーク、報告書文化は10年単位で変わらない構造的特性
- 外資がこれらを理解し対応するコストは、市場規模に見合わない
- ただし、これは「外資が参入しない」理由であると同時に、「市場が小さい」理由でもある

### 5.2 「さくらインターネットが同じことをしたら？」の詳細分析

**さくらインターネットはAIXSにとって最大のMoat破壊者である。** その理由:

1. **ISMAP認証済み + ガバメントクラウド正式認定（2026年3月）:** セキュリティ・信頼性面でAIXSを凌駕
2. **ABCI 3.0の計算資源提供:** 公的研究基盤との関係が既に構築済み
3. **高火力シリーズ（H100/H200/B200）:** GPU計算資源を自社保有で提供
4. **AI事業推進室の設立（2025年8月）:** AIインフラ関連事業を集約し、スピード感ある事業開発体制
5. **生成AIプラットフォーム「さくらの生成AIプラットフォーム」（2025年5月開始）:** 推論用途のLLM実行基盤
6. **TOP500ランキング49位「さくらONE」:** スパコン性能も実証

**さくらがAIXSの領域に参入する確率:** 30-50%（2年以内）

**さくらが参入しにくい理由:**
- さくらのコアビジネスはIaaS/インフラであり、研究者向けSaaS（実験管理、ワークフロー自動化、再現性パッケージ）は「別のレイヤー」
- さくらの組織文化はインフラ運用であり、研究ドメインの深い理解を持つ人材が限定的
- さくらのAI事業推進室は「推論基盤」「企業向けAI」に注力しており、「研究者向けR&Dプラットフォーム」は優先度が低い可能性
- しかし、さくらのパートナー戦略強化（2026年1月報道）は、AIXSのようなSaaS企業との連携を模索している可能性もある

**さくらが参入した場合のAIXS防御策:**
1. **さくらの「パートナー」になる:** さくらが直接研究者向けSaaSを構築するのではなく、AIXSがさくらのインフラ上で動作する「推奨パートナー」として位置づけられれば、むしろさくらのガバメントクラウド認定を活用できる
2. **研究ドメインの深さで差別化:** さくらが追随できない「実験設計→GPU計算→論文再現性パッケージ」のフルサイクルUXに集中
3. **マルチクラウドを維持:** さくら依存を避け、ABCI、mdx、商用クラウドとの統合を同時に維持することで、さくら一社との関係に依存しないポジションを取る

## 7. 結論

### AIXSが構築すべきMoatの核心

AIXSのMoatは**単一の強力な障壁ではなく、複数の中程度の障壁の積層**に依拠すべきである。

```
┌─────────────────────────────────────────────────┐
│  Layer 5: 文化的Moat（最も深い、最も持続的）       │
│  ・根回し文化への適応                              │
│  ・技術職員への対応                                │
│  ・学会ネットワーク                                │
├─────────────────────────────────────────────────┤
│  Layer 4: 市場構造Moat                             │
│  ・年度予算対応                                    │
│  ・科研費費目分類                                  │
│  ・共同利用体制                                    │
├─────────────────────────────────────────────────┤
│  Layer 3: エコシステムMoat                         │
│  ・学認IdP連携                                    │
│  ・ABCI API統合                                   │
│  ・GENIAC/J-Startupへの参画                       │
├─────────────────────────────────────────────────┤
│  Layer 2: 規制Moat                                 │
│  ・ISMAP認証インフラ上での動作                     │
│  ・個人情報保護法準拠                              │
│  ・研究データガバナンス                            │
├─────────────────────────────────────────────────┤
│  Layer 1: プロダクトMoat（最も浅い、最も早く崩壊） │
│  ・マルチクラウドGPUオーケストレーション           │
│  ・実験管理UX                                     │
│  ・再現性パッケージ                                │
└─────────────────────────────────────────────────┘
```

**各Layer単独では持続しないが、5層を積み重ねることで、いずれの競合も完全には模倣できない「複合Moat」が形成される。** これは「幅の広い浅い堀」ではなく、「層の深い堀」である。

### 最後の警告

このMoat分析の前提は「日本市場が$5-20M ARRの天井を持つニッチ市場」であるという既存分析の結論と整合的である。5層のMoatを構築しても、日本市場のTAMの制約は変わらない。Moatの目的は「日本市場での確固たるポジションを確立し、$5-20M ARRの黒字ビジネスを持続的に運営すること」であり、「ユニコーンになること」ではない。

ユニコーンを目指す場合は、Layer 2（規制Moat）をEU/APACに横展開するグローバル戦略が必要だが、これはPhase 3（M24-M36+）以降の検討事項であり、現時点で追求すべきではない。
