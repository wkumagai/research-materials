**作成日:** 2026-03-28
**分析者:** SaaS財務モデリング専門家
**対象:** AIXS経営チーム
**前提:** ToC（個人・小規模チーム）向けGPU+R&D統合プラットフォーム

## 2. 需要予測（保守的）

### 2.1 TAMからSOMへの絞り込み

```
TAM（総市場規模）: GPU-as-a-Service市場
  = $5.8B（2025年）→ $26.6B（2030年、狭義）

SAM（AI研究者向けToC）:
  = TAM × 15%（AI研究者セグメント）× 30%（ToC個人・小規模チーム）
  = $5.8B × 15% × 30% = $261M

SOM_日本（日本市場、初年度獲得可能）:
  = SAM × 5%（日本市場シェア）× 0.5-2%（新規参入のシェア）
  = $261M × 5% × 0.5-2%
  = $65K - $261K/年
  = $5.4K - $21.8K/月（初年度）

SOM_グローバル（グローバル、初年度）:
  = SAM × 0.1-0.5%
  = $261K - $1.3M/年
  = $21.8K - $109K/月（初年度）
```

**基本ケースの初年度目標: MRR $8K-$15K（月間）= ARR $96K-$180K**

これは日本市場SOM下限の$65Kを確実に超え、グローバルSOM中央値$780Kの12-23%に相当する。

## 6. プラン改善提案

### 6.1 粗利率の問題と対策

**根本的な問題:**
GPU再販モデル（Phase 1）では、GPUコストがMRRの55-65%を占め、全体粗利率が30-40%に留まる。SaaS業界の健全な粗利率（70-80%）との乖離が大きい。

**対策1: GPU時間の適正化（本レポートで実施済み）**
- 各プランのGPU包含時間を20-40%削減
- Free: 20hr据え置き、Starter: 50→30hr、Pro: 200→120hr、Team: 500→250hr、Power: 600→400hr
- 結果: 各プランの粗利率が20-30%ポイント改善

**対策2: プラットフォームSaaS料金の分離（中期検討）**
```
現行: プラン月額 = GPU時間 + プラットフォーム機能（込み）
提案: プラン月額 = GPU従量課金 + プラットフォーム月額（$19-$49/月）

例:
  Proユーザーの場合:
    現行: $99/月（H100 15hr込み）→ 粗利41%
    提案: $49/月（プラットフォーム、粗利90%）+ GPU従量$3.50/hr × 15hr = $52.50
    合計: $101.50 → ブレンド粗利58%
```
- ただしこの変更は価格の複雑化とユーザー体験の悪化をもたらすため、PMF達成後に検討

**対策3: 年額契約の推進**
```
年額割引: 20%（月額の10ヶ月分）
例: Pro年額 = $99 × 10 = $990/年（月$82.50相当）
メリット:
  - churn率を月7.2%→年40%に改善（月額churnの年率は60%だが年契約は40%以下）
  - 前払いキャッシュフロー改善
  - LTV向上
コスト:
  - 20%の収益減少
  - 実質粗利はGPU利用パターンに依存（年間一括購入で利用の平準化が期待できる）
```

**対策4: 超過課金の戦略的活用**
- 超過課金はMRRの15-25%を占める（M12以降）
- H100超過は粗利29%と低いが、超過利用はアップセルシグナル
- **月2回以上のH100超過利用ユーザーに上位プランを自動推奨**する仕組みで、超過課金→固定収入への転換を促進

**対策5: 自社インフラ移行のタイムライン（粗利の根本的改善）**
```
Phase 1（M1-M12）: 再販モデル
  - GPU粗利: 30-40%
  - 設備投資: $0
  - リスク: 低（需要変動に柔軟対応）

Phase 2（M13-M24）: ハイブリッド（一部自社GPU購入）
  - 条件: MRR $20K+、稼働率予測精度70%以上
  - 投資: H100 4台 × $25K = $100K（NEDO補助金活用）
  - GPU粗利: 45-55%
  - 損益改善: 月$3K-$5K

Phase 3（M25-M36）: 主に自社インフラ
  - 条件: MRR $50K+、安定ユーザーベース
  - 投資: H100 16台 × $25K = $400K（Series Aまたは自己資金）
  - GPU粗利: 55-70%
  - 損益改善: 月$10K-$15K
```

### 6.2 Proプラン（$99）の粗利改善

**問題:** 元設計のH100 25hr含むProプランは粗利27%。改定後も41%で目標（50%+）に未達。

**追加改善策:**
1. **利用パターン分析に基づく最適化**: H100を実際に使うPro利用者は50%未満と想定。A100利用が大半なら実効粗利は高い
2. **T4/A100のスポットインスタンス活用**: 調達コストを20-30%削減可能
3. **プラットフォーム機能のティアリング**: 自動HP探索やマルチGPU分散はアドオン化（+$20/月）も検討

### 6.3 Powerプラン（$299）の粗利改善

**問題:** 元設計のH100 100hrは粗利10%（実質赤字リスク）。改定後は38%。

**追加改善策:**
1. **GPU時間の実効利用率**: Powerユーザーの全GPU時間消費率は70-85%と想定。未消費分は粗利100%
2. **Power→カスタムへの移行パス**: 月$1,000+のヘビーユーザーにはカスタム見積もり（直接営業）で個別最適化
3. **HPC・量子のクロスセル**: GPUだけでなくHPC・量子シミュレータの利用料で追加粗利を確保（HPC/量子の粗利は80%以上）

### 6.4 プラン体系全体の改善まとめ

| 改善項目 | 実施時期 | 粗利への期待効果 | リスク |
|---|---|---|---|
| GPU包含時間の適正化 | **即時（本レポート反映済み）** | +10-20pp | ユーザー不満の可能性 |
| 年額契約割引の導入 | M3-M6 | churn改善でLTV向上 | 短期MRR20%減 |
| 超過→アップセル自動推奨 | M6-M9 | MRR 5-10%改善 | 低い |
| スポットGPU活用 | M6-M12 | GPU原価15-25%削減 | 安定性リスク |
| プラットフォーム料金分離 | M12-M18（PMF後） | ブレンド粗利+10-15pp | 価格複雑化 |
| 自社GPU購入（少量） | M12-M15 | GPU粗利+15-20pp | 設備投資リスク |
| 自社インフラ本格移行 | M24-M30 | GPU粗利+25-35pp | 大規模設備投資 |
