## 4. プラン設計と価格戦略（ToC = 個人・小規模チーム向け）

### 4.1 ToC向けペルソナ定義

#### ペルソナ1: AI/ML研究者（大学院生〜ポスドク）

| 項目 | 詳細 |
|---|---|
| **誰か** | 情報系・理工系の修士/博士課程学生、またはポスドク研究員。25-35歳。大学・研究機関所属 |
| **何をしたいか** | 学会論文向けの実験（LLM fine-tuning、ViT学習、強化学習、分子シミュレーション等）。A100/H100で数日〜数週間のジョブ実行 |
| **今どうやっているか** | 研究室のローカルGPU（RTX 3090/4090、時々A100）、ABCI（ポイント制：1,000pt = ¥220,000）、Google Colab Pro（$9.99/月、A100で約7時間/100CU）、Kaggle無料GPU（P100/T4、30hr/週） |
| **月額予算感** | 自費: $0-50/月。研究費: $100-500/月。科研費/GENIAC等の外部資金がある場合: $500-2,000/月 |
| **ペインポイント** | ABCIはキュー待ち時間が長い（数時間〜数日）。Colabは実行時間制限・GPU不安定。研究室GPUは他メンバーと共有で空かない。実験管理がローカル（W&B, MLflow手動セットアップ）で再現性確保が手間 |
| **AIXSでの解決策** | 即時GPU確保 + 実験管理統合。ABCIポイントとAIXSクレジットの併用による柔軟なリソース選択。実験メタデータ・再現性の自動記録。論文用の実験結果export機能 |

### 4.2 プラン体系の設計

#### 設計原則

1. **価格帯の位置づけ**: GPU特化プロバイダ（RunPod $1.99-$2.69、Modal $3.95）とハイパースケーラー（GCP $10.98、Azure $12.29）の間の「中間層」に価格を設定し、統合R&Dプラットフォームとしての付加価値で上位の実効価値を提供
2. **課金の透明性**: 隠れコスト（egress、ストレージIOPS等）を排除し、予算予測可能性を最大化
3. **プログレッシブ・バリュー**: 無料→個人→Pro→チームと段階的に価値と予算を拡張する導線

**価格根拠:**
- Kaggle: P100/T4 30hr/週（≈120hr/月）→AIXSは20hr/月で控えめだが、より高性能なL4も選択可能
- Google Colab無料: T4 15-30hr/週だが不安定・中断あり→AIXSは20hr保証付き
- HuggingFace: ZeroGPU（H200）25分/日（≈12.5hr/月）→AIXSは20hr/月でやや上
- Modal: $30/月無料クレジット（H100で約7.5hr相当）→AIXSはT4/L4で20hr（低スペックだがより長時間）
- Lightning AI: 15クレジット/月（GPU数時間相当）→AIXSはより明確な時間枠を提供
- **推定コスト**: T4スポット原価 $0.20-0.40/hr × 20hr = $4-8/月/ユーザー。転換率3-5%で有料ユーザーのLTVが$400+であればCAC回収可能

### 4.4 課金モデル

#### 各課金モデルの比較分析

| モデル | メリット | デメリット | 先行事例 |
|---|---|---|---|
| **従量課金** | 利用量に完全比例で公平。バースト利用に最適。初期コミットメント不要 | 予算予測が困難。月末の請求ショック（bill shock）リスク。収益のボラティリティが高い | Modal（秒課金）、RunPod（秒課金）、AWS EC2 on-demand |
| **サブスクリプション** | 収益予測可能（MRR安定）。解約率が低い。顧客の予算管理が容易 | GPU利用量にかかわらず固定コスト。未使用分の罪悪感→解約。GPU稼働率が低いと粗利悪化 | Colab Pro（$9.99/月）、Lightning AI Pro（$50/月） |
| **クレジット制** | 柔軟性（GPU種を跨いで利用可能）。サブスクと従量課金のハイブリッド。心理的に「使い切り」動機が働く | クレジット→実時間の換算が直感的でない場合がある。繰り越し・失効ルールの設計が複雑 | GCP（CUD）、ABCI（ポイント制、1pt = ¥220）、Lightning AI（クレジット制） |
| **ハイブリッド（基本料+従量）** | 基本料で最低MRR確保 + 従量でアップサイド。顧客は基本枠内で計画的に利用し、超過分は必要時のみ | 料金体系がやや複雑。基本枠の設定が難しい（少なすぎると不満、多すぎると粗利低下） | AWS Savings Plans、Lambda（予約+on-demand）、HuggingFace（サブスク+推論従量） |

#### AIXSに最適なモデル: ハイブリッド（クレジット包含型サブスクリプション + 従量超過課金）

**推奨理由:**

1. **MRR安定性**: サブスク基本料により、スタートアップとしての収益予測可能性を確保。投資家への説明力も向上

2. **柔軟性**: クレジット制により、GPU種（T4→H100）を跨いだ利用が可能。ペルソナ3（コンペ参加者）のようにバースト利用するユーザーにも対応

3. **アップサイド**: 従量超過課金により、ヘビーユーザーからの追加収益を確保。Power ペルソナ（ペルソナ7: 自動研究、ペルソナ8: バイオ研究）のGPU利用量は月によって大きく変動

4. **心理的設計**: 月額のクレジット包含分は「使わないと損」の動機を生み、プラットフォームへの習慣化を促進。超過課金は「必要な時だけ追加」で心理的抵抗が少ない

5. **競合差別化**: Modal（純従量課金）との差別化。予算管理のしやすさでModalに不満を持つユーザーを吸収

## 5. なぜAIXSが競合に対して優位なのか（競合優位性の詳細論証）

### 5.1 各直接競合に対する差別化ポイント

#### vs RunPod（最安価格帯）

**RunPodの強み**: H100 SXM $2.69/hr（community）、H100 PCIe $1.99/hr（市場最安級）、秒課金、30+ GPU SKU、500K+開発者、$120M+ ARR

**AIXSが勝つシナリオ:**
1. **実験管理が必要な研究者**: RunPodは「GPUを貸す」だけ。実験管理、ハイパーパラメータ探索、結果比較はすべてユーザー自身で構築する必要がある。W&B/MLflowを自前でセットアップする手間（推定: 初回2-4時間、メンテナンス月2-3時間）をAIXSが内蔵機能で解消
2. **GPU + HPC/量子の統合ワークロード**: RunPodはGPUオンリー。分子シミュレーション→ML学習のパイプラインは不可能
3. **チーム利用**: RunPodにはチーム管理、プロジェクト別コスト分離、ロール管理が存在しない（2026年3月時点）
4. **信頼性重視のワークロード**: Community Cloudは中断リスクあり。Secure Cloudは$3.89/hr（H100）でAIXSのPro超過料金$3.50/hrと同等

**AIXSが負けるシナリオ:**
1. **純粋なGPU時間の最大化**: 予算$100でH100を最大時間使いたい場合、RunPod community ($2.69) は37hr、AIXS Pro は25hr。RunPodが48%多い
2. **Spot/Preemptible需要**: RunPodはH100 Spot $1.50/hrを提供。AIXSが同等のSpot制度を持たない場合、コスト競争で劣位
3. **推論デプロイ特化**: RunPod Serverless はカスタムモデルの推論デプロイに特化。AIXSのR&Dフォーカスと異なる用途

**総合判定**: AIXSはRunPodの「上位互換」を目指すのではなく、「RunPodでGPUを借りて手作業で実験管理している人に、統合体験を提供する」ポジショニング。GPU単価で勝つ必要はない。

#### vs Modal（開発者体験）

**Modalの強み**: 最高水準のDX（Python SDK、秒課金、冷起動1-2秒）、サーバーレス（ゼロアイドル）、$30/月無料枠、$25Kスタートアップクレジット

**AIXSが勝つシナリオ:**
1. **長時間学習ジョブ**: Modalはサーバーレスに最適化されており、数日にわたるfine-tuningジョブには向かない（タイムアウト制約）。AIXSは永続インスタンスで長時間学習に対応
2. **実験管理統合**: Modalは「コードを実行する」だけ。実験のメタデータ管理、結果比較、ハイパーパラメータ探索はユーザー責任
3. **非Pythonワークロード**: ModalはPython SDKに限定。C++/Fortran科学計算（HPC）、量子シミュレータ等は対象外
4. **予算予測**: Modalの従量課金は月末まで総額が不明。AIXSのクレジット包含型サブスクは予算管理が容易

**AIXSが負けるシナリオ:**
1. **推論サービング**: Modalのサーバーレス推論（冷起動1-2秒、自動スケール）は推論デプロイに最適。AIXSがこのレベルのサーバーレス推論を提供するには時間がかかる
2. **DX至上のユーザー**: Modalの `@app.function(gpu="H100")` デコレータの簡潔さは、インフラを意識したくない開発者にとって最強。AIXSが同等以上のDXを実現するまで、この層は奪えない
3. **スケーリングの自動化**: Modalの自動スケールは0→1000 GPUまでシームレス。AIXSがスケーリング機能を構築するには投資が必要

**総合判定**: Modalは「最高のDXでGPUコードを実行する」プラットフォーム。AIXSは「R&Dライフサイクル全体を管理する」プラットフォーム。競合関係よりも「Modal上でコードを実行 → AIXSで実験を管理・比較」の補完関係も検討すべき。ただしToC向けでは一気通貫体験が重要なため、長期的にはModal同等のDXを目指す必要がある。

4. **チーム機能**: Lambdaはチーム管理、コスト分離等のコラボレーション機能が限定的

**AIXSが負けるシナリオ:**
1. **大規模クラスター学習**: Lambdaの1-Click Cluster（H100 $2.76/hr、16-2000+ GPU）はAIXSが提供できない規模
2. **Lambda Stack のシンプルさ**: SSH接続 → すぐにPyTorch/TensorFlowが動く環境は、複雑な機能不要のユーザーにとって最適
3. **在庫安定性**: $1.5B調達によるGPU大量確保。小規模AIXSでは在庫確保で劣る可能性
4. **B200 $4.99/hr**: 最新GPU の価格リーダーシップ

**総合判定**: Lambda Labsは「最もシンプルなGPUクラウド」。AIXSは「シンプルGPU + R&D統合」で差別化。LambdaユーザーのうちW&B/MLflowを別途使っている層がAIXSのターゲット。

**AIXSが負けるシナリオ:**

**総合判定**: HuggingFaceとは競合するより統合すべき。「HFのモデルをAIXSでfine-tune → HFにデプロイ」の導線を構築。HFのGPU計算の高さ（$10/hr H100）に不満を持つユーザーをAIXSに吸収。

**総合判定**: Colab/Kaggleは「入口」、AIXSは「次のステップ」。Colab/Kaggleの無料GPUで限界を感じたユーザーを自然にAIXSに誘導する導線が重要。「Colabの実行時間切れ、もう3回目ですか？ AIXSなら安定GPU保証」のようなメッセージング。

### 5.2 「GPU+α」の価値の定量化

#### レイヤー別の価値積み上げ

```
┌─────────────────────────────────────────────────────┐
│ Layer 4: GPU + 異種資源統合（HPC + 量子 + ラボ機器） │  追加WTP: +100-200%
│   「GPUとHPCと量子を一つのパイプラインで」           │  提供者: AIXSのみ
├─────────────────────────────────────────────────────┤
│ Layer 3: GPU + ワークフロー統合                      │  追加WTP: +40-60%
│   「実験→評価→デプロイの自動化」                    │  競合少数
├─────────────────────────────────────────────────────┤
│ Layer 2: GPU + 実験管理                              │  追加WTP: +20-40%
│   「実験の自動記録・比較・再現」                     │  W&B, MLflow等は別製品
├─────────────────────────────────────────────────────┤
│ Layer 1: GPU単体利用                                 │  基準価格
│   「GPUを借りてコードを実行」                        │  RunPod, Lambda, Modal等
└─────────────────────────────────────────────────────┘
```

#### 各レイヤーでの定量的価値比較

##### Layer 1: GPU単体利用

| 指標 | AIXS | 競合ベスト | AIXS vs 競合 |
|---|---|---|---|
| H100 1GPU-hr | $3.50（Pro超過） | RunPod $2.69（community） | -23%（RunPodが安い） |
| H100 保証付き 1GPU-hr | $3.50（Pro超過） | RunPod Secure $3.89 | +10%（AIXSが安い） |
| H100 25hr/月（Pro相当） | $99（サブスク込み） | Lambda $99.75 | ≈同額（AIXSは追加機能込み） |

**結論**: GPU単体ではAIXSに価格優位性はない。RunPod communityには勝てない。保証付きGPUではやや優位。

##### Layer 2: GPU + 実験管理

| 構成 | AIXS | 競合組み合わせ | 差額 | 時間節約 |
|---|---|---|---|---|
| H100 25hr + 実験管理（無制限） | $99（Pro） | Lambda $100 + W&B Free（200hr制限） | -$1 | W&Bセットアップ2-4hr省略 |
| H100 25hr + 実験管理（チーム向け） | $99（Pro） | Lambda $100 + W&B Teams $50/人 × 3 | -$151 vs $250 | 統合管理で月3-5hr省略 |
| H100 25hr + HP探索10回 | $99（Pro） | Lambda $100 + Optuna自前セットアップ | -$1 + 時間差 | HP探索インフラ構築5-10hr省略 |

**追加WTP推定**: +$20-40/月（月額の20-40%増）
- W&B Pro: $50/月 → ユーザーは実験管理に$50/月の支払い意思がある
- MLflow自前運用コスト: EC2 t3.medium $30/月 + メンテナンス時間2hr/月 × $50/hr = $130/月の実質コスト

##### Layer 3: GPU + ワークフロー統合

| 構成 | AIXS | 競合組み合わせ | 差額 | 時間節約 |
|---|---|---|---|---|
| H100 25hr + 実験管理 + ワークフロー自動化 | $99（Pro） | Lambda $100 + W&B $50 + Prefect Cloud $50 | $99 vs $200 | Prefect/Airflowセットアップ8-16hr省略 |
| H100 75hr + パイプライン自動化 | $299（Power） | Lambda $299 + W&B $50 + Prefect $50 + カスタム統合 | $299 vs $399+ | 統合構築20-40hr省略 |

**追加WTP推定**: +$40-60/月（月額の40-60%増）
- ワークフローオーケストレーション（Prefect Cloud $50/月、Airflow自前運用$100+/月）の代替価値
- エンジニア時間の節約: 月4-8時間 × $80-150/hr = $320-1,200/月の機会費用

##### Layer 4: GPU + 異種資源統合（HPC + 量子 + ラボ機器）

| 構成 | AIXS | 競合組み合わせ | 差額 | 備考 |
|---|---|---|---|---|
| GPU + HPC + 量子（1パイプライン） | $299（Power） | Lambda + ABCI + IBM Quantum + 自前統合 | $299 vs $500-1,500+ | 統合構築は推定100+hr |
| 分子シミュレーション→ML→実験設計 | $299（Power） | AWS ParallelCluster + SageMaker + 自前統合 | $299 vs $800-3,000+ | AWSの複雑性は著しい |

**追加WTP推定**: +$100-300/月（月額の100-200%増）
- 異種資源統合は**AIXSの唯一無二の提供価値**。競合が存在しない
- このレイヤーの価値を体験したユーザーのスイッチングコストは極めて高い（パイプライン再構築に100+時間）
- ペルソナ7（自動研究）、ペルソナ8（バイオ・材料）が主要ターゲット

#### 累積価値のサマリー

| レイヤー | GPU単価での競合力 | 追加WTP | AIXS提供価値 | 競合状況 |
|---|---|---|---|---|
| L1: GPU単体 | 劣位（RunPod -23%） | 基準 | コモディティ | 激戦（30+プロバイダ） |
| L2: +実験管理 | 同等 | +20-40% | 差別化開始 | 一部（W&B統合のSaaS） |
| L3: +ワークフロー | 優位（統合で-50%） | +40-60% | 明確な差別化 | 極少数 |
| L4: +異種資源 | 独占的 | +100-200% | 唯一無二 | 競合なし |

**戦略的示唆**: AIXSはLayer 1で戦うべきではない。Layer 2-3で顧客を獲得し、Layer 4で圧倒的なロックインと高WTPを実現するのが最適戦略。
