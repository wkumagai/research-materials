## 4. 大手は構造的な理由で参入できない

### 主張：AWS、Google、Azureは「R&D統合プラットフォーム」を作る能力はあるが、作るインセンティブがない。これは一時的な事情ではなく、大企業の構造的制約に起因する。

#### 根拠4-1：ハイパースケーラーは自社GPU稼働率の最大化が至上命令

AWS、Google、Azureの経営KPIは「自社データセンターのGPU稼働率」である。彼らが他社のGPU（RunPod、さくら、ABCI）にジョブをルーティングする機能を作るインセンティブは**ゼロ**。

- AWSは2026年に**100万台以上のNVIDIA GPU**をデプロイ予定。加えて自社チップTrainium/Inferentiaへの移行を推進。他社GPUへの分散は**自社チップ戦略と根本的に矛盾**する。
- Googleは独自TPU（Ironwood、第7世代、前世代比5倍のピーク性能）を開発中。Vertex AIで**TPUロックイン**を推進している。
- MicrosoftはAzure Quantum等の個別サービスは持つが、**Vertex AIとAzure Quantumの統合ワークフロー**すら未実装。社内の組織構造（プロダクトチームの縦割り）が統合を阻害する。

マルチクラウドGPUオーケストレーション（AWS + GCP + RunPod + さくら + ABCIを横断的に最適配分）は、**ハイパースケーラーが絶対にやらないこと**であり、AIXSの根本的な差別化。

## 5. AIXSの防御可能性（Moat）は構造的に高い

### 主張：AIXSは3つの独立したMoatを重ねて構築でき、これらは時間の経過とともに指数関数的に強化される。

#### 根拠5-1：スイッチングコスト -- 実験データの蓄積が「データ重力」を生む

クラウドプラットフォームの最強のMoatは**データ重力（Data Gravity）**。研究者がAIXS上で蓄積するデータは時間とともに移行不可能になる:

- 研究ラボが1年間で蓄積する実験データ: **数TB〜数十TB**（モデルチェックポイント、実験ログ、データセット）
- このデータを他プラットフォームに移行するEgressコスト: **$数千〜$数万**（AWSのEgress料金$0.09/GB × 10TB = $900、100TBなら$9,000）
- 移行に必要な時間: ペタバイト級のデータ移動は**数日〜数週間**
- 加えて、実験メタデータ（ハイパーパラメータ履歴、モデル比較ダッシュボード、チームのナレッジグラフ）は**プラットフォーム固有フォーマット**で蓄積

クラウドのスイッチングコストは「platform dependency economics, developer network effects, and switching costs」の三重構造であり、**蓄積データ量の二乗に比例して移行コストが増大**する。AIXSが意図的にリッチな実験メタデータ構造を設計すれば、100回の実験後には事実上のロックインが成立する。

#### 根拠5-2：ネットワーク効果 -- GPU供給者 x 研究者の両面市場

マーケットプレイスの最も強力なMoatは両面ネットワーク効果:

- **供給側**: GPUプロバイダーが増える → 価格競争力↑・可用性↑ → 研究者が増える
- **需要側**: 研究者が増える → ワークフローテンプレート・実験ナレッジが蓄積 → プラットフォーム価値↑ → さらに研究者が増える
- **間接ネットワーク効果**: 研究者コミュニティが大きくなる → コラボレーション機会↑ → 学術コミュニティでの標準化 → 新規参入者のデフォルト選択になる

AWSの成功がこの力学を証明している。「more apps and services are built natively on AWS because that's where the customers and the data are. The ecosystem of solutions attracts more customers and developers who build more apps that generate more data.」AIXSのR&D特化版ネットワーク効果は、汎用クラウドよりも**垂直的であるがゆえに密度が高く、構築が速い**。

**定量的目安**:
- ユーザー1,000人 × 平均100テンプレート/ユーザー = **10万ワークフローテンプレート**のライブラリ
- この規模のライブラリを新規参入者がゼロから構築するには**3-5年**
- 臨界質量到達後の成長は指数関数的（Metcalfeの法則: ネットワーク価値 ∝ n^2）

#### 根拠5-3：技術的先行者優位 -- 異種資源統合は18-36ヶ月の参入障壁

「GPU + HPC + 量子 + ラボ装置」の統合コントロールプレーンは、技術的に**極めて困難**:

1. **プロトコルの異質性**: GPU（CUDA/ROCm）、HPC（Slurm/PBS）、量子（Qiskit/Cirq/Pennylane）、ラボ装置（各社独自プロトコル）を統一APIで抽象化する必要がある
2. **ハイブリッドワークフローの複雑性**: 「GPUで学習 → HPCでシミュレーション → 量子で最適化 → ラボ装置で検証」を単一DAGで記述・実行する技術は前例がない
3. **AIエージェント操作インターフェース**: LLMが自然言語で実験を指示・実行するAPIの設計は、ドメイン知識とAI知識の両方が必要

先行者が18-36ヶ月で構築するこの技術基盤を、後発者が追い越すには**同等以上の期間**が必要。しかも先行者はその間にもデータ蓄積・ネットワーク効果の強化を続けるため、実質的な追い越しは**5年以上のタイムラグ**になる。

これは「技術的複雑性 × データ蓄積 × ネットワーク効果」の**三重の堀（Triple Moat）**であり、1つが突破されても他の2つが防御する構造。
