**作成日:** 2026-03-28
**分析者:** VC-grade戦略分析
**対象:** AIXS経営チーム
**前提:** AIXSは「GPU + HPC + 量子 + ロボティクス + 実験装置」を単一コントロールプレーンで統合し、AIエージェント操作可能なR&Dコンピュートプラットフォームを構築する少人数スタートアップ

## 分析1: AIXSは事業として成り立つか

### 1.1 市場機会の定量化

#### AI GPU Cloud市場規模（2025-2030）

複数の調査機関のデータを統合すると、GPU-as-a-Service（GPUaaS）市場は以下の成長軌道にある:

| 年 | 市場規模（USD） | 出典 |
|---|---|---|
| 2024 | $21B | Analysys Mason |
| 2025 | $5.7B - $8.2B | Fortune BI / MarketsandMarkets（定義差による幅） |
| 2026 | $7.3B - $12B | 各社推計の中央値 |
| 2030 | $26.6B - $134B | MarketsandMarkets（狭義） / Analysys Mason（広義） |
| 2032 | $49.8B | Fortune Business Insights |

**注:** 市場定義により大幅な差異がある。狭義のGPUaaS（純粋なクラウドGPUレンタル）は$26.6B（2030年）、広義（AIインフラ全体を含む）は$134B。AIXSが狙う「R&Dコンピュートプラットフォーム」はこの中間に位置する。

データセンターGPU市場（ハードウェア含む）はさらに大きく、2030年に$77-81B規模。

#### 研究R&D向け計算基盤のTAM/SAM/SOM推定

```
TAM（総市場規模）: $80B - $134B（2030年）
  = GPUaaS全体 + HPCクラウド + 研究自動化プラットフォーム + 量子クラウド

SAM（獲得可能市場）: $8B - $15B（2030年）
  = R&D特化コンピュート（学術研究 + 企業R&D + 創薬 + 材料科学）
  = GPUaaS市場の約10-15%がR&D用途と推定
  ※ 根拠: AI研究者・R&Dチームはクラウドコンピュート利用者の10-15%
  ※ 加えてHPCクラウド市場（$4B、2025年）の研究利用分

SOM（短期獲得可能市場）: $50M - $200M（2030年）
  = 日本市場のR&Dコンピュート + 初期海外展開
  = 日本のAIインフラ予算（METI ¥1.23T = $7.9BのうちR&D向け~10%）の0.5-2%
  ※ ABCI利用者、大学・研究所のGPU需要、国内企業R&D部門
```

#### 「GPU + 異種R&D資源統合」未開拓セグメントの規模推定

現時点で「GPU + HPC + 量子 + ラボ装置」を統合提供するプレーヤーは**存在しない**。これは以下の理由から:

1. **各資源が異なるベンダー・プロトコルで管理されている**: GPUはCUDA/ROCm、HPCはSlurm/PBS、量子はQiskit/Cirq、ラボ装置は各社独自プロトコル
2. **顧客セグメントが異なる**: GPU→ML研究者、HPC→物理シミュレーション、量子→量子情報研究者、ラボ装置→実験科学者
3. **既存プレーヤーにとって隣接市場への拡張インセンティブが低い**: CoreWeaveやLambdaはGPUの需要対応だけで成長率80%+を達成しており、異種資源への拡張は優先度が低い

**未開拓セグメント推定規模:**

- R&D統合プラットフォーム（GPU + 実験管理 + ワークフロー自動化）: **$2B - $5B**（2030年）
  - 根拠: MLOpsプラットフォーム市場（$4B、2025年 → $18B、2030年）の一部 + GPUaaS市場のR&D向けプレミアム層
- 異種コンピュート統合（GPU + HPC + 量子）: **$500M - $1.5B**（2030年）
  - 根拠: HPC-as-a-Service市場（$7B、2030年予測）の異種統合需要 + 量子クラウド市場（$2B、2030年予測）の統合需要
- ラボ装置統合: **$200M - $800M**（2030年）
  - 根拠: Lab-as-a-Service市場の成長（Emerald Cloud Lab $92M調達、Strateos同規模）

**AIXSが狙う統合セグメント合計: $2.7B - $7.3B（2030年）**

### 1.2 事業成立の条件

#### 最低限の月間売上ターゲット

少人数スタートアップとしての存続に必要な最低月間売上:

| フェーズ | チーム規模 | 月間バーンレート | 必要月間売上（粗利60%想定） | 必要月間売上（粗利30%想定） |
|---|---|---|---|---|
| Pre-seed（0-6ヶ月） | 2-3人 | ¥150万（$10K） | $17K | $33K |
| Seed（6-18ヶ月） | 3-5人 | ¥500万（$33K） | $55K | $110K |
| Series A（18-36ヶ月） | 8-15人 | ¥2,000万（$133K） | $222K | $443K |

**損益分岐のための最低MRR: $55K - $110K（Seed段階、月間バーン¥500万想定）**

#### 必要ユーザー数とARPU想定

3つの顧客セグメントを想定:

| セグメント | 月間ARPU | 必要ユーザー数（MRR $110K達成） | 想定プロファイル |
|---|---|---|---|
| 個人研究者 | $200 - $500 | 220 - 550 | 大学院生、ポスドク。月20-50 GPU-hours |
| 研究ラボ/チーム | $2,000 - $10,000 | 11 - 55 | 大学研究室、企業R&Dチーム。月200-1,000 GPU-hours |
| 機関・企業 | $10,000 - $100,000 | 1 - 11 | 国立研究所、製薬企業。月1,000-10,000 GPU-hours |

**現実的なターゲット:** 研究ラボ/チーム 20-30 + 個人研究者 50-100 = MRR $100K-$150K

これは達成可能な数字である。日本国内だけでもABCI利用研究者は数千人おり、さくらDOKの利用者も増加中。MLプラットフォーム機能がないことへの不満は顕在化している。

#### GPU調達コスト vs 販売価格のユニットエコノミクス

**H100 SXM 1GPU-hourあたりのコスト構造（2026年3月時点）:**

| モデル | 調達コスト/GPU-hr | 販売価格/GPU-hr | 粗利/GPU-hr | 粗利率 |
|---|---|---|---|---|
| **再販モデル（RunPod等から調達）** | $1.99 - $2.69 | $3.50 - $5.00 | $0.81 - $2.31 | 23% - 53% |
| **再販モデル（AWS/Lambda等から調達）** | $2.76 - $3.99 | $4.50 - $6.00 | $0.51 - $2.01 | 11% - 40% |
| **自社インフラ（H100購入、3年償却）** | ~$0.85 - $1.45 | $3.50 - $5.00 | $2.05 - $4.15 | 59% - 83% |
| **ハイブリッド（一部自社 + スポット活用）** | ~$1.20 - $2.00 | $3.50 - $5.00 | $1.50 - $3.80 | 43% - 76% |

**注:**
- 自社インフラの$0.85-$1.45/GPU-hrは、H100 1台$25,000-$30,000を3年償却（$0.95-$1.14/hr@100%稼働率）+ 電力・冷却・ネットワーク（$0.20-$0.30/hr）+ データセンターホスティング（$0.10-$0.20/hr）で算出
- 実際の稼働率は60-80%が現実的であり、100%稼働は非現実的。稼働率70%の場合、自社インフラコストは$1.21-$2.07/GPU-hrに上昇
- SemiAnalysisの分析では、「GPUの主要コストは資本コスト（$7,025/月）であり、ホスティングコスト（$1,871/月）を大幅に上回る」

#### 粗利率シミュレーション

**シナリオA: 純粋再販モデル（Phase 1推奨）**

```
前提:
- RunPod/Lambda等から$2.50/GPU-hrで調達
- $4.00/GPU-hrで販売（プラットフォーム付加価値込み）
- プラットフォームSaaS料金: 月額$50-$200/ユーザー（GPU費用とは別）

GPU粗利: ($4.00 - $2.50) / $4.00 = 37.5%
SaaS粗利: 90%（ほぼ限界費用ゼロ）
ブレンド粗利: 40-55%（SaaS比率による）
```

**シナリオB: 自社インフラモデル（Phase 3以降）**

```
前提:
- H100 32台を$800K-$960Kで購入（3年償却）
- 稼働率70%
- $3.50/GPU-hrで販売

調達コスト: ~$1.50/GPU-hr（稼働率70%調整後）
GPU粗利: ($3.50 - $1.50) / $3.50 = 57.1%
SaaS粗利: 90%
ブレンド粗利: 60-70%
```

**シナリオC: ハイブリッドモデル（Phase 2推奨）**

```
前提:
- 基本需要の60%を自社インフラ（$1.50/GPU-hr）
- ピーク需要の40%をスポット/オンデマンド（$2.00-$2.69/GPU-hr）
- $4.00/GPU-hrで販売

加重調達コスト: $1.50 × 0.6 + $2.20 × 0.4 = $1.78/GPU-hr
GPU粗利: ($4.00 - $1.78) / $4.00 = 55.5%
SaaS粗利: 90%
ブレンド粗利: 58-68%
```

**結論:** 初期は再販モデル（粗利40-55%）で資本効率を重視し、PMF達成後に自社インフラへ移行（粗利60-70%）するのが最適。

#### 損益分岐点の試算

**Phase 1（再販モデル、チーム3-5人）:**

```
固定費（月間）:
- 人件費: ¥400万（$27K）— 3-5人のエンジニア（日本の水準）
- インフラ（非GPU）: ¥30万（$2K）— サーバー、SaaS、ドメイン等
- その他: ¥70万（$4.7K）— オフィス、法務、雑費
合計: ¥500万/月（$33K/月）

損益分岐に必要な売上（粗利45%想定）:
$33K / 0.45 = $73K/月

内訳:
- GPU時間: 月間15,000 GPU-hours × $4.00/hr = $60K
  → ユーザー30人 × 平均500 GPU-hours/月
- SaaS料金: 30ユーザー × $100/月 = $3K
- プラットフォーム手数料等: $10K
合計: $73K/月
```

**損益分岐までの期間: 12-18ヶ月（Seed資金$500K-$1M前提、月間$33Kバーン）**

これはSaaS業界のシード→Series A間の中央値（19ヶ月）と整合的。

### 1.3 成立しない可能性のあるシナリオ

#### シナリオ1: GPU価格がさらに下落し差別化困難

**リスク評価: 中-高**

- **現状:** H100価格は18ヶ月で$8+→$2.50-4.00に下落（約50-70%減）。Blackwell B200の登場でH100はさらに下落圧力。2026年初頭にはHBM不足で一時10%反発したが、長期トレンドは下落
- **最悪ケース:** H100が$1.50/hr以下、B200が$3.00/hr以下に。再販マージンが10%以下に圧縮
- **発生確率:** 30-40%（2年以内）。GPU供給増加（TSMC CoWoS増産、AMD MI300X、Intel Gaudi3）により加速可能性
- **回避策:**
  1. **GPU価格ではなくプラットフォーム価値で課金する**: GPU-hourの再販マージンではなく、SaaS月額（$100-$500/ユーザー）をプライマリ収益源にする。GPUは原価またはマージナルコストで提供し、「実験管理 + ワークフロー自動化 + AIエージェント統合」のプラットフォーム価値で課金
  2. **GPU agnosticアーキテクチャ**: NVIDIA、AMD、Intel、カスタムASICを横断的にサポートし、最も費用対効果の高いGPUに動的にルーティング
  3. **上位レイヤーの価値創出**: 「GPU時間からインサイト獲得までの時間（Time-to-Insight）を50%短縮」のような、GPUコモディティ化の影響を受けない価値指標を確立

#### シナリオ2: ハイパースケーラーが統合プラットフォーム機能を追加

**リスク評価: 中**

- **現状:** AWS SageMaker、GCP Vertex AI、Azure AMLは既にMLプラットフォーム機能を持つが、R&Dワークフロー統合（実験計画→コンピュート→分析→論文）は不十分。HPCや量子との統合はさらに弱い
- **最悪ケース:** AWSが「SageMaker for Research」のような製品を出し、ABCI的な研究ポータルを構築
- **発生確率:** 20-30%（3年以内）。ハイパースケーラーはエンタープライズ市場に注力しており、研究R&Dはニッチすぎて優先度が低い
- **回避策:**
  1. **ハイパースケーラー上で動くメタレイヤーとして設計**: AWS/GCP/Azure上で動作するマルチクラウドオーケストレーターとして位置づければ、ハイパースケーラーは競合ではなくインフラ基盤になる
  2. **研究者コミュニティとの密着**: ハイパースケーラーの弱点はドメイン知識。研究者のワークフロー（仮説→実験→解析→論文）を深く理解し、UXに反映
  3. **異種資源統合というユニークバリュー**: ハイパースケーラーは自社GPUの稼働率最大化が最優先であり、RunPodやさくらのGPUにジョブをルーティングする動機がない

#### シナリオ3: 市場がコモディティ化して薄利多売

**リスク評価: 中**

- **現状:** 2025年に300+の新規プロバイダがH100クラウド市場に参入。Vast.aiのP2Pモデルでは$1.49/hrまで下落。中国の算力券による30+都市のGPU過剰供給（稼働率30%）が象徴的
- **回避策:**
  1. **GPUレイヤーでは競争しない**: AIXSの売り物はGPUではなく「R&D生産性プラットフォーム」。GPUはインプットの一つに過ぎない
  2. **バーティカルSaaS化**: 創薬R&D、材料科学R&D、AI安全性研究など特定ドメインに特化したワークフローテンプレートを提供し、ドメイン知識で差別化
  3. **課金モデルの転換**: GPU-hour課金から「研究プロジェクト単位」「実験回数課金」「インサイト課金」へ移行し、コモディティ化したGPU価格から切り離す

#### シナリオ4: 資金調達できずスケールできない

**リスク評価: 中-高（日本のスタートアップとして）**

- **現状:** GPU/AIインフラ分野のグローバルVC投資は巨額（CoreWeave $5.1B売上、Lambda $1.5B調達、Modal $111M調達）。一方、日本のVCエコシステムは小規模（2024年の日本VC投資額は約$3-4B、米国$194Bの2%以下）
- **回避策:**
  1. **公的資金の活用**: NEDO DTSU（最大1億円、費用の2/3補助）、JST CREST/PRESTO、J-Startup認定を早期に獲得
  2. **日本+シンガポール+グローバルVCの組み合わせ**: Coral Capital、Global Brain、DNX Ventures（日本）+ Sequoia Southeast Asia（シンガポール）
  3. **顧客収益による自己資金化**: Seed段階で5-10の研究ラボを獲得し、MRR $100K+を達成してからSeries Aを調達（バリュエーション交渉力が大幅に向上）
  4. **GPU調達を自社で行わない「アセットライト」モデル**: 初期は再販モデルで資金需要を最小化

#### シナリオ5: 技術的複雑性により開発が遅延

**リスク評価: 高**

- **現状:** GPU + HPC + 量子 + ロボティクス + ラボ装置の統合は技術的に非常に野心的。各資源のAPI/プロトコルが異なり、統一コントロールプレーンの構築は複雑
- **回避策:**
  1. **段階的統合**: Phase 1はGPUのみ、Phase 2でHPC追加、Phase 3で量子、Phase 4でラボ装置。一度に全てを統合しない
  2. **既存OSSの活用**: Kubernetes（オーケストレーション）、MLflow（実験管理）、Argo Workflows（ワークフロー）、Ray（分散コンピュート）を基盤に、統合レイヤーのみを自社開発
  3. **抽象化レイヤーの設計**: 各資源を統一的なインターフェースで扱えるリソース抽象化レイヤー（Resource Abstraction Layer）を設計し、新資源の追加を容易にする

## 分析2: Moat（競争優位性の堀）の構築

### 2.1 少人数スタートアップが構築可能なMoat

#### a) ネットワーク効果（両面市場: GPU供給者 x 利用者）

**実現可能性: 中（長期的に構築可能）**

- **メカニズム:** 研究者が増える → 実験データ・ワークフローテンプレートが蓄積 → プラットフォームの価値が上がる → 新規研究者が参加。GPU供給者が増える → 価格競争力が上がる → 研究者が増える
- **課題:** 初期のコールドスタート問題。鶏と卵。RunPod（50万開発者）やModal（ARR $50M）が既にネットワーク効果を持ち始めている
- **AIXSの戦略:**
  - 「R&D特化」という垂直セグメントに絞ることで、汎用プラットフォームとは異なるネットワーク効果を構築
  - 研究者同士のコラボレーション機能（共有実験ログ、再現性保証）でR&Dコミュニティ特有のネットワーク効果を生む
  - 初期は日本の研究コミュニティ（ABCI利用者、学会ネットワーク）に集中し、局所的な密度を高める
- **Moat強度: ★★★☆☆**（構築に2-3年。ただし一度構築されれば強固）

#### b) スイッチングコスト（データ・実験ログ・ワークフローの蓄積）

**実現可能性: 高（初期から意図的に設計可能）**

- **メカニズム:** 研究者がAIXS上で100回の実験を実施 → 実験ログ、ハイパーパラメータ履歴、モデルアーティファクト、可視化ダッシュボードが蓄積 → 他プラットフォームへの移行コストが指数的に増加
- **データ重力（Data Gravity）:** MLflowの思想とは逆に、「コンピュートがデータの近くに移動すべき」というデータ重力の法則を活用。ペタバイト級のデータセット移動は数日かかりEgressコスト$数千-数万
- **AIXSの戦略:**
  - 実験メタデータのリッチな構造化（仮説→手法→結果→考察の研究ライフサイクル全体をキャプチャ）
  - プロジェクト横断的な知識グラフ（「過去に似た実験をした研究者Bの結果」を自動推薦）
  - 再現性パッケージ（実験環境 + データ + コードを完全再現可能な形でアーカイブ）
  - 論文執筆支援（実験結果から自動的にFigure/Table生成、LaTeX出力）
- **Moat強度: ★★★★☆**（最も早期に構築可能で、最も堅実なMoat）

#### c) 独自データ（利用パターン、最適配置アルゴリズム）

**実現可能性: 中-高（データの蓄積に時間が必要）**

- **メカニズム:** 大量の研究ワークロード実行データ → 最適なGPU配置、バッチサイズ、学習率スケジュール等をAIが推薦 → 研究者の生産性が向上 → さらにデータが蓄積
- **AIXSの戦略:**
  - ワークロードプロファイリングデータの蓄積（「このタイプの実験はH100よりA100の方がコスト効率が良い」）
  - 研究パターンの類型化（「トランスフォーマー学習」「強化学習」「分子シミュレーション」等のワークロード特性を学習）
  - 障害予測・自動復旧の精度向上
- **Moat強度: ★★★☆☆**（データ量に比例。1,000ユーザー×1年で有意なデータセット）

#### d) ブランド/コミュニティ（研究者コミュニティでのポジション）

**実現可能性: 高（少人数でも戦略的に構築可能）**

- **メカニズム:** 研究者は口コミとピアレビューで意思決定する。学会発表、OSSコントリビューション、技術ブログがブランドを形成
- **AIXSの戦略:**
  - 主要AI学会（NeurIPS, ICML, ICLR, ACL）でのスポンサー・ワークショップ開催
  - トップ研究者への無料アクセス提供（「AIXS Research Fellow」プログラム）
  - 再現性コンテスト（Reproducibility Challenge）のホスト
  - OSSツールの公開（実験管理CLI、ワークフローテンプレート）
  - 日本の学会（JSAI, 情報処理学会, 電子情報通信学会）での強いプレゼンス
- **Moat強度: ★★★☆☆**（低コストで開始可能。ただし競合も同様の戦略を取れる）

#### e) 規模の経済（GPU調達ボリューム）

**実現可能性: 低（少人数スタートアップには不向き）**

- **メカニズム:** 大量のGPU調達 → ボリュームディスカウント → 低価格提供 → ユーザー増
- **現実:** CoreWeave（$5.1B売上、100K+ GPU）、Lambda（$520M売上）と比較して、AIXSが調達ボリュームで競争するのは非現実的。NVIDIAは大口顧客を優遇しており、小規模バイヤーは不利
- **AIXSの戦略:** 規模の経済はMoatとして追求**しない**。代わりにマルチクラウドアービトラージ（複数プロバイダの最安値をリアルタイムで選択）で実質的なコスト削減を実現
- **Moat強度: ★☆☆☆☆**（追求すべきでない）

#### f) 技術的優位性（異種資源統合コントロールプレーン）

**実現可能性: 高（AIXSのコアコンピタンスとして最重要）**

- **メカニズム:** GPU + HPC + 量子 + ラボ装置を統一APIで操作可能なコントロールプレーンは、技術的に複雑で構築に時間がかかる。先行者優位が大きい
- **AIXSの戦略:**
  - リソース抽象化レイヤー（Resource Abstraction Layer）の設計・実装
  - AIエージェント操作インターフェース（LLMが自然言語で実験を指示・実行できるAPI）
  - ハイブリッドワークフロー（「GPUで学習 → HPCでシミュレーション → 量子で最適化 → ラボ装置で検証」を単一ワークフローで記述）
- **技術的詳細:**
  - コンテナオーケストレーション: Kubernetes拡張（Custom Resource Definitions for 各資源タイプ）
  - ワークフローエンジン: Argo Workflows / Prefect上に研究ドメイン特化DAGを構築
  - リソーススケジューラ: コスト・性能・可用性のマルチ目的最適化
- **Moat強度: ★★★★★**（最も強力だが構築に最も時間がかかる。18-36ヶ月で有意な優位性）

#### g) 制度的障壁（政府認証、データ主権）

**実現可能性: 中-高（日本市場では特に有効）**

- **メカニズム:** 日本政府の研究データはISMAP認証クラウド上での処理が必要。GDPR/APPI（個人情報保護法）対応。国家安全保障関連の研究は国内インフラ必須
- **AIXSの戦略:**
  - ISMAP認証の早期取得（さくらインターネットのISMAP認証インフラ上で動作させることで迅速化可能）
  - 文科省・経産省との関係構築（ABCI 3.0のフロントエンドパートナーとして提案）
  - ソブリンAI需要への対応（国内データセンターでの処理保証）
  - GENIAC（生成AI基盤モデル開発プロジェクト）への参画
- **Moat強度: ★★★★☆**（日本市場では非常に強力。海外展開時にはEU GDPR/AI Act対応が新たなMoatに）

### 2.3 競合が真似しにくいポイント

#### なぜModalやRunPodがAIXSと同じことをやらないのか

**1. 事業フォーカスの相違**

| 要素 | Modal | RunPod | AIXS |
|---|---|---|---|
| コアビジネス | サーバーレスGPUコンピュート | GPU IaaS/PaaS | R&Dコンピュートプラットフォーム |
| 主要顧客 | ソフトウェアエンジニア | AI開発者全般 | 研究者・R&Dチーム |
| 収益モデル | GPU-hour課金 | GPU-hour課金 | SaaS + GPU-hour |
| ARR | ~$50M | ~$120M | - |
| 成長戦略 | 開発者体験の深化 | 価格競争力 + 規模拡大 | R&Dワークフロー統合 |

Modal（ARR $50M、バリュエーション$2.5B交渉中）は「最高の開発者体験のサーバーレスGPU」に集中している。R&D実験管理は彼らのコアコンピタンスではなく、**追加するインセンティブが低い**。研究者向け機能を追加しても、既存の50万開発者ユーザーベースへの価値向上が限定的。

RunPod（ARR $120M、$22M調達のみで効率的成長）は**価格リーダーシップ**に集中している。$1.99/hrのH100 PCIeを提供する彼らが、粗利率を下げるR&Dプラットフォーム機能を追加する合理性は低い。

**2. 組織能力の相違**

- ModalとRunPodのエンジニアリングチームはインフラ/DevOps人材で構成。研究ドメインの知識（実験設計、統計解析、論文ワークフロー）を持つ人材がいない
- 研究者のペインポイントを理解するには、自身が研究者であるか、研究者と密接に協働した経験が必要
- AIXSの創業チームが研究バックグラウンドを持つことが決定的な差別化要素

**3. 市場規模の相違**

- ModalとRunPodが狙う「AIアプリケーション開発」市場は$50B+。彼らにとってR&D市場（$8-15B）はTAMの15-30%に過ぎず、戦略的優先度が低い
- AIXSにとってはR&D市場がTAMの100%であり、全リソースを集中投下可能

#### なぜAWSやGCPがR&D統合をやらないのか

**1. ビジネスモデルの構造的制約**

- ハイパースケーラーはインフラ利用量（GPU-hour、ストレージGB、ネットワーク帯域）で課金する。利用量を**最大化**するインセンティブ
- AIXSのR&D効率化ツールは研究者のGPU利用を**最適化**（=削減）する。GPU利用を減らすツールをAWSが積極提供する動機がない
- 例: 「このハイパーパラメータ設定ならA100で十分、H100は不要」というレコメンデーションはAWSの売上を減らす

**2. 顧客セグメントのミスマッチ**

- AWSの平均顧客はエンタープライズ企業（年間$100K-$10M+のクラウド利用）
- R&D研究ラボの平均年間クラウド利用は$5K-$50K。AWSのセールスチームが対応するには小さすぎる
- 大学/研究所の調達プロセス（入札、年度予算制度）はエンタープライズとは根本的に異なる

**3. 技術的アプローチの相違**

- ハイパースケーラーは汎用性を重視（数百万の顧客に対応）。R&D特化UXの構築は汎用プラットフォームの設計哲学と矛盾
- SageMakerは「MLエンジニア向けMLOps」であり、「研究者向けR&D管理」ではない。仮説管理、実験ノート、論文執筆支援はSageMakerのロードマップに存在しない
- 異種資源統合（HPC + 量子 + ラボ装置）はAWSのビジネスモデル外。AWSは自社GPU/Trainium/Inferentiaの利用促進が最優先

#### AIXSだけが有利な構造的理由

1. **100%のリソースをR&D市場に集中**: Modal/RunPodはAIアプリ開発、AWSはエンタープライズIT。AIXSだけがR&Dに全集中
2. **マルチクラウドの中立性**: 特定のGPUプロバイダに依存しないため、常に最適なコスト/性能のGPUにルーティング可能。AWS/GCPは自社リソースの利用を最大化するインセンティブがあり、マルチクラウドは構造的にできない
3. **日本のR&D市場への地理的優位性**: ABCI、さくら、GPUSOROBANなど日本固有のインフラとの統合。ISMAP対応。日本語サポート。日本の研究文化（年度予算制度、学会ネットワーク）への理解
4. **AIエージェント操作というタイミング**: 2026年はAIエージェントの実用化元年。研究プロセスのAIエージェント操作（AI Scientist v2、FutureHouse Robin等）は今まさに市場が立ち上がっており、既存プレーヤーはまだ対応していない
5. **異種資源統合の先行者優位**: GPU→HPC→量子→ラボ装置という統合ロードマップを持つプレーヤーは世界に存在しない。最初に統合コントロールプレーンを構築すれば、技術的先行優位は2-3年持続

#### 技術スタック選定の方針

| レイヤー | 選定 | 理由 |
|---|---|---|
| オーケストレーション | Kubernetes + Custom CRDs | 業界標準。GPUプロバイダの多くがK8s互換 |
| ワークフローエンジン | Argo Workflows or Prefect | OSSで成熟、DAG定義が容易 |
| 実験管理 | MLflow互換API（自社実装） | デファクト標準との互換性 + 独自拡張 |
| フロントエンド | Next.js + TypeScript | 高速開発、SSR、React生態系 |
| バックエンド | Python (FastAPI) + Go（性能クリティカル部分） | ML生態系との親和性 + 高性能 |
| データベース | PostgreSQL + TimescaleDB（メトリクス） | 実績、拡張性 |
| オブジェクトストレージ | S3互換（MinIO or クラウドS3） | アーティファクト保存の標準 |
| 認証 | Clerk or Auth0 | Build不要、SAML/SSO対応 |
| インフラ | Terraform + Pulumi | マルチクラウドIaC |

#### Buy vs Build の判断基準

| コンポーネント | 判断 | 理由 |
|---|---|---|
| GPUプロビジョニング | **Build**（コア） | 差別化要素。マルチクラウドオーケストレーションは独自ロジック |
| 実験管理 | **Build on OSS** | MLflowのAPIを参照しつつ、研究ワークフロー特化の拡張を実装 |
| 認証/課金 | **Buy** | Clerk/Stripe。差別化要素ではない |
| モニタリング | **Buy/OSS** | Grafana + Prometheus。GPU利用のカスタムダッシュボードのみBuild |
| CI/CD | **Buy** | GitHub Actions。差別化要素ではない |
| ワークフローエンジン | **OSS + 拡張** | Argo/Prefectを基盤に研究ドメインDSLを構築 |
| フロントエンド | **Build** | UXが差別化要素。研究者のメンタルモデルに合ったUI |

- 創業者の自己資金 + 日本政策金融公庫の創業融資（最大$50K-$100K）で賄える可能性あり

**ただし、以下のリスク:**
- 6ヶ月でMVPと初期ユーザーを獲得できなかった場合の資金枯渇
- GPU価格のスプレッド（調達価格と販売価格の差）が薄い場合のキャッシュフロー圧迫
- 競合の動きが速い場合、資金制約がスピードのボトルネックに

#### シード資金の必要額推定

**推奨シード調達額: ¥5,000万 - 1億円（$330K - $670K）**

| 法務/会計/設立費用 | $20K - $40K | 6% |
| バッファ（予備費） | $30K - $130K | 19% |

**18ヶ月のランウェイを確保するなら: $500K - $1M**

#### 初期売上での自己資金化可能性

**シナリオ: Phase 2終了時点（6ヶ月後）**

```
楽観シナリオ: MRR $30K、月間バーン $33K → バーンマルチプル 1.1x → ほぼ自己資金化
現実シナリオ: MRR $15K、月間バーン $33K → バーンマルチプル 2.2x → 追加資金必要
悲観シナリオ: MRR $5K、月間バーン $33K → バーンマルチプル 6.6x → 戦略見直し必要
```

SaaS業界のベンチマーク: シード段階のバーンマルチプル中央値は3-5x。2x以下なら優秀。

## 総合判定

### AIXSは事業として成り立つか？

**結論: 条件付きで成立する（確信度: 65-75%）**

**成立の条件:**
1. **GPU再販ではなくプラットフォーム価値で課金する**: GPU-hourのマージンだけでは成立しない。SaaS月額 + GPUマージンのブレンドで粗利50%+を確保
2. **日本市場から開始する**: 国内R&Dコンピュート市場にMLプラットフォーム機能を持つプレーヤーが不在というホワイトスペースを活用
3. **12ヶ月以内にMRR $50K+を達成する**: PMFの早期検証。達成できなければピボットまたは撤退判断
4. **異種資源統合を段階的に実現する**: GPU→HPC→量子→ラボ装置の順で、各段階でユーザー価値を証明

**最大のリスク:** 技術的野心と初期実行のバランス。全てを同時に構築しようとせず、GPUオーケストレーション + 実験管理のMVPに集中すること。

**最大の機会:** AIエージェント操作可能なR&Dプラットフォームは、2026年の今がまさにタイミング。AI Scientist v2やFutureHouse Robinが示すように、研究プロセスのAI自動化は実用段階に入りつつあり、そのインフラ基盤を提供するプレーヤーは世界的に不在。

### Moat構築の実現可能性

**結論: 18-36ヶ月で意味のあるMoatを構築可能（確信度: 60-70%）**

- **最も堅実なMoat:** スイッチングコスト（実験データ・ワークフローの蓄積）★★★★☆
- **最も強力なMoat:** 技術的優位性（異種資源統合コントロールプレーン）★★★★★
- **最も早期に構築可能なMoat:** ブランド/コミュニティ + 制度的障壁（日本市場）★★★☆☆ + ★★★★☆

### 少人数スタートアップとしての実行可能性

**結論: 実行可能だが、厳格なスコープ管理が不可欠（確信度: 70-80%）**

- 3人チーム、$500K-$1Mのシード資金、18ヶ月のランウェイで、MVPからPMF検証まで到達可能
- 成功の鍵は「やらないことの明確化」。GPU価格競争、自社インフラ構築、全資源同時統合は初期にやるべきでない
- RunPod（ARR $120M、$22M調達のみ）が示すように、アセットライトモデルで高効率な成長は可能
